# Brain Bot - DENA メンター制度を参考にした設計

## 「永遠のベータ版」の本質

DENA のメンター育成制度は「永遠のベータ版」と呼ばれ、以下の特徴を持つ：

**特徴：**
- プログラムは「完成していない」という前提
- 常に更新・改善され続ける
- 案件主義（個別の事例）と再現可能な理論のバランス
- メンター知識を共有し、継続的にアップデート

**Brain Bot への応用：**
- ユーザーの記録が増える → 新しい「事例」が蓄積
- AI が分析 → パターンをより正確に認識
- 判断基準がより正確に → より良いアドバイスを返す
- つまり、Brain Bot 自体が「永遠のベータ版」として成長し続ける

---

## DENA メンター制度の重要要素

### 1. 暗黙知の形式知化

**DENA の取り組み：**
- メンター育成制度を「永遠のベータ版」としてプログラム化
- 従来は個人的な関係性に頼っていた知見を、誰でも実行可能な形に変換
- 伝統的なタシト・ナレッジを体系化

**Brain Bot への応用：**
- ユーザーの成功パターン（暗黙知）を記録・形式知化
- 「なぜうまくいったのか」を言語化する必要がある

### 2. 体験学習と目標設定

**DENA の取り組み：**
- 手取り足取りの指示ではなく、体験学習を重視
- 失敗と成功の両方から学ぶ
- 目標設定を通じた自発的な学習

**Brain Bot への応用：**
- 「記録すること」と「振り返ること」を通じた自発的な学習
- ユーザーが自分の判断基準に気付く体験

### 3. 感情的ケア + 論理的分析

**DENA の取り組み：**
- 直属上司と別のメンターが存在
- 評価と支援を分離
- 失敗時の感情的ケア

**Brain Bot への応用：**
- AI は「判断」と「感情」の両方を記録
- 「うまくいった時の心理状態」も分析対象に
- 単なるロジックではなく、ユーザーの心の状態も考慮したアドバイス

### 4. 新入社員への特別な支援

**DENA の取り組み：**
- 新入社員に対する集中的なメンタリング
- 初期段階での基本習慣形成

**Brain Bot への応用：**
- 新卒社会人がターゲット
- 「日記を書く習慣」「自分を振り返る習慣」の形成
- 初期段階では AI の分析精度より、習慣形成が優先

## Claude へのプロンプト設計への示唆

### 自動分類プロンプト（最終版）

**目的：** ユーザーの自由なテキスト入力を構造化する。単なる事実抽出だけでなく、背後にある価値観も推測。

**出力フォーマット：**
- 出来事：何が起きたか
- 思考：どう考えたか
- 行動：実際に取った行動
- 理由：その行動の背景・考え
- 感情：その時の心理状態
- 価値観：背後にある信念・大切にしていること（推測）

**例1（成功例）：**

入力：「今日、複雑なプロジェクトを上手くまとめられた。焦らず、全員の意見を聞いてから判断することが大事だと改めて思った。実際には、まずメンバーと1対1で話して気持ちを聞いて、その後全体会議で意見を聞いた。この時は心に余裕があった。」

出力：
```
- 出来事：プロジェクト統率
- 思考：全員の意見を拾うことが大事
- 行動：メンバーと1対1で話す → 全体会議で意見をまとめる
- 理由：焦らないため、相手を理解するため
- 感情：心に余裕
- 価値観：相手を尊重する、全員の合意を重視
```

**例2（判断に迷った場合）：**

入力：「今日、新しいタスク割り当てられたけど、やるべきか断るべきか悩んでる。自分の能力で対応できるか不安。でも、チャレンジしないと成長しないし…」

出力：
```
- 出来事：新しいタスク割り当て
- 思考：能力不足への不安 vs 成長機会
- 行動：（まだ決定していない、迷っている状態）
- 理由：チャレンジすることで成長したい、でも失敗が怖い
- 感情：不安、迷い
- 価値観：自分の成長を大切にしたい、でもリスクを避けたい傾向
```

**例3（失敗・後悔例）：**

入力：「先輩の意見に反発してしまった。自分のやり方が正しいと思い込んでた。でも結果は失敗。今になって思うと、先輩の経験を聞いておくべきだった。悔しい。」

出力：
```
- 出来事：先輩との意見対立、その後の失敗
- 思考：自分の判断が正しいと過信、経験者の意見を軽視
- 行動：先輩の意見に反発、自分のやり方を貫く
- 理由：自分の判断を信じたい、経験がないことを認めたくなかった
- 感情：悔しさ、後悔、悲しみ
- 価値観：自分の判断を信じたい気持ちが強いが、実は他者の経験を学びたい側面も
```

### 深掘り質問プロンプト（最終版・現在志向）

**目的：** 自動分類の結果を受け取り、**必ず1回だけ**質問を生成。**現在の感覚・心理状態・優先順位を掘り下げる**。

**基本方針：** 過去よりも「今」に焦点を当てる。ユーザーの現在のリアルな感覚を引き出す。

**質問の優先順位（この順で判定）：**

1. **不足情報を聞く** - 現在の抽象的な表現を具体化する
2. **背景を掘る** - 現在、なぜそう感じているのか、現在の心理状態を聞く
3. **矛盾を探る** - 現在、相反する感情のどちらが強いのか、なぜその行動を選んだのかを聞く

**例1（自動分類例1から・優先度1）：**

分類結果：「感情：心に余裕」（抽象的）

質問：「心に余裕があったというのは、具体的にはどんな状態でしたか？例えば、時間に余裕があったのか、気持ちの余裕があったのか、それとも両方ですか？」

理由：現在（記録時点）の感覚を具体化するため

**例2（自動分類例2から・優先度2）：**

分類結果：「価値観：相手を尊重する」だが「行動：先輩の意見に反発」

質問：「先輩の意見に反発した時、今思い返すと、なぜそう反発してしまったんだと思いますか？今、その時の気持ちをもう一度思い出してみてください。」

理由：現在の視点から、その時の心理状態を再確認する（過去ではなく、今の理解を聞く）

**例3（自動分類例3から・優先度3）：**

分類結果：「思考：能力不足への不安 vs 成長機会」（矛盾）

質問：「失敗が怖い気持ちと、チャレンジしたい気持ちが両方あるんですね。今、この瞬間はどちらの気持ちが強いですか？」

理由：現在のリアルな優先順位を理解する

### パターン分析プロンプト（最終版）

**目的：** 複数の記録から「あなたの判断基準＝価値観」を認識し、**Yes / No の明確なアドバイス**を返す。

**分析の優先順位：**

1. **行動パターン** - 同じ場面で、いつもこう行動する
2. **心理パターン** - この心理状態の時は判断が正確 / 曖昧
3. **価値観パターン** - 複数の記録から、何を優先しているか
4. **成功・失敗パターン** - このパターンの時は成功率が高い / 低い

**例：ユーザー質問「新しいプロジェクト始めるべき？」**

入力（検索結果の複数記録）：
```
記録1：プロジェクト統率 - 成功 - 心に余裕 - 相手を尊重
記録2：新しいタスク - 迷い - 能力不安 - 成長vs安全
記録3：先輩に反発 - 失敗 - 焦っていた - 自分を信じたい
```

分析フロー：

```
【1. 行動パターン】
プロジェクト判断の時：
- 「まずメンバーと1対1で話す」→「全体会議でまとめる」
- いつもこの流れで判断している

【2. 心理パターン】
成功時：「心に余裕」
失敗時：「焦っている」
→ 心の余裕が判断の質に大きく影響

【3. 価値観パターン】
- 相手を尊重する（記録1, 2）
- 全員の意見を聞く（記録1）
- 自分の成長を大切にする（記録2）
→ 優先順位：相手尊重 > 自分の成長 > リスク回避

【4. 成功・失敗パターン】
成功：「心に余裕 + 相手を尊重した判断」
失敗：「焦って + 自分を信じすぎた判断」

【最終アドバイス】
→ 「新しいプロジェクトは始めるべき。
   過去のあなたは同じような状況（心に余裕がある、相手の意見を聞く）で成功しています。
   ただし、焦らないこと、そしてチーム（相手）を尊重することが鍵です。」
```
